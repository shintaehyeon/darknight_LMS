# 다크나이트 (Dark Knight) - LMS 미디어 익스트랙터 MVP

LMS(학습 관리 시스템)의 동영상 시청 및 스트리밍 환경에서 미디어를 식별하고 관리할 수 있도록 설계된 크롬 확장 프로그램 프로토타입(MVP)입니다.

---

## 🛠️ 제공하는 핵심 기능 (MVP 기준)
1. **지능형 네트워크 스니퍼 (`background.js`)**
   - 백그라운드 네트워크 레이어에서 흐르는 미디어 스트림 주소를 실시간 식별합니다.
2. **DOM 파싱 엔진 (`content.js`)**
   - 현재 페이지에 포함된 `<video>`, `<source>`, `<iframe>` 태그의 구성 요소를 읽어내어 플레이어 정보를 수집합니다.
3. **사용자 인터페이스 (`popup.html` / `popup.css` / `popup.js`)**
   - 공상과학(Sci-Fi) HUD 다크 스타일 테마의 팝업창을 통해 수집된 미디어 목록을 직관적으로 확인하고 주소 복사 및 상세 모달 기능을 제공합니다.

---

## ⚠️ 미해결 과제 및 아키텍처 한계 (Technical Debt & Limitations)
현재 구현된 프로토타입은 **로컬 클라이언트 독립형 모델**로 동작하므로 다음과 같은 기술적 한계를 안고 있습니다:

1. **클라이언트 사이드 검증 우회 취약성**:
   - 다운로드 횟수 제한이나 크레딧 소모 로직이 로컬 스토리지에만 존재할 경우, 크롬 개발자 도구(Console)나 확장 프로그램 삭제 후 재설치를 통해 사용자가 손쉽게 제한을 무력화할 수 있습니다.
2. **미디어 서버 측 엄격한 CORS/CSP 규제**:
   - 특정 미디어 호스트가 강력한 웹 보안 규칙(CSP 및 동일 출처 정책)을 설정해 놓은 경우, 브라우저 환경에 따라 `TypeError: Failed to fetch` 혹은 `403 Forbidden` 연결 거부가 발생할 수 있습니다.
3. **세션 쿠키 및 네트워크 헤더 가로채기(DNR)의 한계**:
   - 브라우저의 SameSite 쿠키 보안 규정과 특권 백그라운드 호출 제한으로 인해, 브라우저 샌드박스를 완벽하게 벗어난 순수 로컬 데이터 수집은 브라우저 업데이트 및 보안 강화에 따라 차단될 수 있습니다.

## 🚀 향후 개발 로드맵 및 SaaS 전환 작업 (Future Tasks for SaaS Development)
상용화 서비스로 안정적이고 보안성 높게 배포하기 위해 반드시 수행해야 할 후속 아키텍처 개발 작업 목록입니다.

### 1. 보안 서버리스 백엔드 구축 (Firebase Auth + Cloud Functions)
- **목적**: 불법 크레딧 복제 차단 및 유저별 권한 인증 중앙 제어.
- **작업 내용**:
  - 확장 프로그램 팝업 내 구글 간편 로그인(Firebase Auth) 탑재.
  - NoSQL 데이터베이스(Firestore)를 구축하여 유저별 크레딧 잔여량과 하루 무료 다운로드 수량 통제.
  - 데이터 차감 로직을 외부 클라우드 함수(API 엔드포인트)로 격리하여 안전성 극대화.

### 2. PG사 결제 연동 (Payment Gateway Integration)
- **목적**: 유저가 다운로드 크레딧을 직접 충전할 수 있는 충전 시스템 마련.
- **작업 내용**:
  - 토스페이먼츠, Stripe 또는 아임포트(Iamport) 연동.
  - 결제 완료 시 웹훅(Webhook)을 통해 Firestore 유저 크레딧을 자동으로 가산해 주는 중간 백엔드 라우터 구현.

### 3. 서버 사이드 프록시 다운로더 (Proxy Downloader) 구축 고려
- **목적**: 확장 프로그램 클라이언트 코드 내의 미디어 수집 메커니즘을 외부에 감추고, 보다 안정적인 네트워크 망 구축.
- **작업 내용**:
  - AWS Lambda 혹은 서버리스 환경에서 수집 요청을 대신 받아 최종 `.mp4` 완성본 바이너리를 컴파일한 뒤, 이를 서명된 보안 임시 URL(Signed URL)로 클라이언트에 전달하는 방식으로 서버-클라이언트 아키텍처 전환.
